# 1. 产品核心概述
「幻界冒险模拟器」是一款面向TRPG/D&D爱好者及文字冒险休闲用户的手机端网页游戏，核心定位为「单人纯文字AI跑团模拟器」。轻量易上手，玩家通过文字输入与AI互动，体验沉浸式冒险，核心优势的是强规则约束性、AI丰富互动边界、游玩便捷性、手机端适配性，开发落地难度低、周期短，具备清晰的后续扩展空间（联网、多人、变现）。
- 核心定位：手机端网页·纯文字·AI跑团TRPG模拟器
- 目标用户：TRPG入门爱好者、文字冒险玩家、碎片化时间手机用户
- 核心亮点：规则强约束、AI担任DM主持游戏进程、游戏过程不脱离、体验不跑偏
- 开发团队：策划（张国玺）、技术（赵阳），核心模块分工明确，落地效率高
# 2. 游戏怎么玩？
游戏全程纯文字互动，无需视觉操作，玩家通过输入文字描述自己的行为，即可完成冒险全流程，AI全程担任“主持人”（DM），负责叙事、判定和反馈，玩法门槛低、沉浸感强。
## 2.1 核心玩法流程
1. 打开网页：无需下载安装，手机浏览器打开即可；
1. 选择模组：从预设的模组中挑选（可理解为“冒险剧本”），每个模组包含独立剧情、地图、敌人等核心内容；
1. 创建角色：选择职业、种族、生成属性等内容、自定义姓名，生成专属角色；
1. 开始冒险：输入指令与AI互动（如“去森林”“攻击敌人”“查看状态”），AI实时反馈场景和结果；
1. 持续推进：完成战斗、探索、解谜等事件，积累经验升级，推进剧情，直至完成模组冒险。

### 2.3.3 行动处理
- 初始距离：遭遇敌人时，由模组预设初始距离（例如“敌人在中程”），AI同步告知玩家；
- 移动操作：玩家可输入“接近”（拉近1个距离层）或“后退”（推远1个距离层），该操作消耗1次移动行动；
- 回合制战斗：玩家输入复合行动指令（例如“接近+攻击”“后退+施法”），AI自动处理先攻判定、战场状态描述，反馈行动结果；
- 地形影响：战场中会随机出现抽象buff/debuff（例如“掩护区：+2 AC”“泥泞地：移动速度降低”），增加战斗策略性**（这部分1.0不开发）**
### 2.3.4 非战斗机制
除战斗外，冒险核心还包含探索、对话、技能检定（就是dnd的技能检定）三大非战斗机制，均由AI基于预设模组推进，玩家通过文字指令触发，例如“探索四周”“与村民对话”“尝试开锁”，AI根据规则判定结果，推进剧情走向。
## 2.4 核心玩法亮点（吸引用户的关键）
- 轻量无负担：纯文字交互，无需下载，离线可玩，适配碎片化时间（通勤、休息等）；
- 沉浸感强：AI生成沉浸式场景描述，玩家自主决策，每一步都影响冒险走向；
- 规则公平：基于D&D 5e简化规则，所有数值、判定由程序固定，无AI自主篡改，兼顾趣味性和公平性；
- 易上手：初始版本开发DND5E基础规则全职业（不含扩展职业），操作仅需文字输入，新手也能快速适应。
# 3. 游戏怎么做？（核心实现逻辑，易懂不晦涩）
本版本（V1.0）为纯前端开发，无需后端服务器，核心依托“前端技术+规则硬编码+本地AI”实现，开发难度低、周期短，可快速落地。核心设计思路为「强规则限制」：**所有数值计算、规则逻辑由传统开发写死，AI仅负责交互推进，不参与任何数值生成和规则制定**，具体实现逻辑及分工如下。
## 3.1 核心分工：AI负责部分 vs 传统开发负责部分（核心设计思路）
明确区分两者职责，是本产品的核心设计亮点，既保证规则公平性，又提升开发效率，具体分工如下：
### 3.1.1 AI负责部分（仅推进、交互、判定反馈，不碰数值/规则）
AI核心定位为“智能主持人（DM）”，所有行为均遵循传统开发写死的规则和预设的模组内容，不自主生成任何数值、奖励或规则外内容，具体负责：
- 剧情与模组推进：解析玩家指令，匹配预设模组的剧情节点，推进主线、支线剧情，生成沉浸式场景和对话描述；
- 战斗判定反馈：接收传统开发（规则引擎）的数值计算结果（如掷骰、伤害、命中），转化为通俗文字描述，反馈给玩家，同时推进战斗流程（如先攻顺序、行动反馈）；
- 对话与交互响应：解析玩家的自然语言指令（如“与村民对话”“探索四周”），匹配对应场景，生成NPC对话、探索结果，响应玩家操作；
- 公共事件库导入：在冒险过程中（如探索支线时），从传统开发预设的公共事件库中，导入适配当前场景的事件模板（如遇敌、谜题），并推进事件流程；
- 状态反馈：将传统开发计算的角色状态（HP、经验、装备），以清晰的文字形式反馈给玩家（如状态面板展示）。
### 3.1.2 传统开发负责部分（规则、数值、技术落地，核心管控）
传统开发（前端+程序）负责所有核心规则、数值计算和技术实现，是游戏公平性和稳定性的核心保障，具体负责：
- 规则硬编码：将D&D 5e简化规则（属性、技能、战斗、升级）通过代码写死，所有判定逻辑（如命中、豁免、技能检定）固定，AI仅可调用，不可修改；
- 数值与奖励管控：所有数值（HP、伤害、经验、金币等）、奖励内容，均由预设模组确定或通过通用事件判定系统（程序写死随机表）生成，AI无权自主生成或修改；
- 技术架构落地：前端界面（交互、布局）、离线缓存（PWA）、本地存档（localStorage）、模组解析等技术实现，确保游戏流畅运行；
- 模组与事件库搭建：制定统一的模组JSON规范，开发模组解析功能，预设核心模组；创建公共事件库（遇敌、谜题、NPC互动等模板），明确事件触发条件和结果；
- 核心系统开发：角色系统（DND5E基础规则全职业、3个种族的属性与能力写死，不含扩展职业）、地图系统（节点逻辑、移动规则）、战斗系统（距离层、地形buff等规则）的开发与管控。
## 3.2 核心技术栈（简单说明，无需深入）
全程采用前端技术开发，无需复杂服务器部署，降低开发和运维成本，具体技术栈如下：
- 前端：HTML5 + JavaScript + CSS3（实现界面、交互和核心逻辑）；
- 离线支持：PWA技术（缓存游戏资源）+ localStorage（本地存档，无需联网）；
- AI：本地轻量LLM或预生成脚本（负责解析指令、生成描述，不依赖云端）；
- 模组解析：原生JS JSON解析（适配预设的“冒险剧本”，可灵活替换）。
## 3.3 核心实现逻辑（3大模块，支撑全游戏运行）
### 3.3.1 规则引擎模块（游戏的“公平基石”，传统开发负责）
核心是将D&D 5e规则简化后，通过代码硬编码实现，所有数值、判定逻辑固定，确保公平性，无需人工干预，AI仅可调用该模块的结果：
- 掷骰系统：虚拟骰子（d20、d6等），随机数由程序生成，支持优势/劣势判定；
- 属性与技能：6个核心属性（力量、敏捷等），修正值计算固定，技能检定逻辑统一；
- 战斗与升级：战斗判定、伤害计算、等级提升均按预设规则执行，AI仅负责反馈结果。
### 3.3.2 交互与AI模块（游戏的“主持人”，AI负责核心交互）
核心负责“理解玩家指令+反馈场景结果”，实现玩家与游戏的顺畅互动，所有行为均遵循传统开发预设的规则和内容：
- 指令解析：识别玩家输入（自然语言或简单命令，如“/status查看状态”）；
- 场景生成：根据玩家指令和模组内容，生成沉浸式文字描述；
- 逻辑调用：触发规则引擎，接收计算结果，转化为通俗文字反馈给玩家。
### 3.3.3 模组与地图模块（游戏的“冒险载体”，传统开发+AI协同）
传统开发负责搭建模块框架和内容预设，AI负责调用和推进，具体分工：
- 传统开发：制定模组规范、开发模组解析功能，预设模组内容（剧情、地图、敌人、奖励），创建公共事件库模板；
- AI：根据玩家指令，调用预设模组推进剧情，导入公共事件库模板，适配场景触发事件；
- 地图系统：传统开发开发地图节点逻辑和移动规则，AI负责展示地图、反馈移动结果和触发相关事件。
## 3.4 开发核心优势（落地保障）
- 纯前端开发，无需后端服务器，开发、部署成本低；
- 核心逻辑硬编码，稳定性高，bug风险低，测试难度小；
- 模组化设计，后续可快速新增职业、冒险剧本，迭代成本低；
- 手机端深度适配，响应延迟≤5秒，体验流畅；
- 分工清晰：AI与传统开发职责边界明确，既保证规则公平，又提升开发和交互效率，契合最初核心设计思路。
