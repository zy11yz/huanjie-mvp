## 二、v1.0.1 核心需求

### 2.1 AI DM Agent 系统（P0）

**目标**：创建一个受规则约束的 AI 游戏主持人，AI 负责叙事和指令解析，但不直接修改游戏状态。

**功能要求**：
1. AI DM 必须遵循预设游戏规则，不能自由发挥超出规则范围
2. 所有判定必须通过规则引擎验证（掷骰、技能检定、战斗计算等）
3. AI DM 负责叙事描述、NPC 对话、剧情推进，但数值判定由系统执行
4. 支持玩家输入自然语言，AI DM 解析后转换为规则动作

**约束条件（强规则系统）**：
- ❌ AI 不能修改角色属性、物品、状态——只能读取
- ❌ AI 不能决定判定结果——只能调用规则引擎
- ❌ AI 不能跳过流程——必须遵循游戏阶段（探索→遭遇→战斗→奖励）
- ✅ 所有 AI 输出必须经过敏感词过滤和规则合规检查

**技术实现架构**：
```
玩家输入 → AI 接口层 → 验证层（中间件） → 规则引擎层 → 返回结果
```

---

### 2.2 故事模组系统（P0）

**目标**：支持预先编写的故事模组，AI DM 在模组框架内主持游戏。

**模组结构规范**：
```
模组/
├── meta.json          # 模组元信息
├── chapters/          # 章节文件夹
├── npcs/              # NPC 定义
├── encounters/        # 遭遇定义
└── items/             # 物品定义
```

**运行时行为**：
1. AI DM 读取当前章节，根据玩家行动推进剧情
2. 遭遇触发时，切换到战斗/检定模式
3. 模组提供"提示"给 AI DM，但不强制玩家选择

---

### 2.3 UI 美术升级（P1）

**目标**：原地迭代 5 个美术版本，找到最佳视觉风格。

| 版本 | 风格方向 | 核心变化 |
|------|----------|----------|
| v1 | 暗黑奇幻 | 深色调、哥特字体、血迹效果 |
| v2 | 东方玄幻 | 水墨风、书法字体、粒子特效 |
| v3 | 赛博朋克 | 霓虹色、科技感、HUD 元素 |
| v4 | 极简现代 | 扁平化、无衬线字体、留白 |
| v5 | 复古像素 | 8-bit 风格、像素字体、CRT 效果 |

## 三、验收标准

### 3.1 AI DM 系统验收
- [ ] AI 无法绕过规则引擎直接修改游戏状态
- [ ] 所有玩家动作都能被正确解析为规则指令
- [ ] AI 叙事与规则判定分离，日志可追溯
- [ ] 响应延迟 ≤ 5 秒

### 3.2 故事模组系统验收
- [ ] 能加载并运行示例模组（约 3000 字）
- [ ] 章节切换流畅，NPC 对话正确触发
- [ ] 支持至少 3 个分支结局

### 3.3 UI 美术验收
- [ ] 5 个版本全部完成并可热切换
- [ ] 性能无显著下降（FPS > 30）
