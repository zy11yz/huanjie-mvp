# 开发计划

这个文档将3月底要开发的内容分模块列出来，供确认工作量使用。

## 核心模块

### dnd基础规则开发-判定系统

1. **投掷系统**
   - 从多个随机范围中，进行正整数的数字随机，对行为的结果进行判定
   - 游戏中，几乎所有的行为，都要遵循此套判定机制

2. **判定系统**
   - 所有的判定，都会调用投掷系统得出一个数字结果，然后根据数字结果决定判定结果
   - 判定行为必然为两个值的对比，即目标难度值和判定值
   - 判定值通常情况为投掷系统获得的投掷值加上从玩家身上所获取的一系列加值
   - 若判定值≥目标难度值，则判定成功，否则判定失败
   - 成功和失败分为多种类型，这部分后续开发
   - 通常情况下，导入的模组中会预设成功和失败的情况
   - 每个可判定内容的可判定次数分为有限和无限，具体情况由模组决定

3. **组合判定和判定队列**
   - 某些判定中，需要多次投掷并进行判定来决定

4. **战斗系统**
   - 战斗系统的基础逻辑基于投掷系统和判定系统进行开发
   - 可以粗暴的理解为，大多数战斗行为，都是使用不同数量的不同骰子进行不同条件的判定
   - 在beta1版本中，只做攻击行为的判定

5. **事件系统**
   - 非战斗的事件统一归纳入事件系统
   - 事件系统是玩家全部需要检定的除战斗行为的统合，例如说钓鱼、采药、过桥等
   - 当玩家触发事件后，调用判定系统进行对应的判定

### dnd基础规则开发-角色系统

1. **角色系统（核心）**
   - 角色是玩家进行游戏的主体
   - 玩家每次在进行游戏时，都需要创建一个角色来进行游戏

2. **基础属性（核心）**
   - 每个属性都是一个值
   - 这些值通常会作为判定系统中的加值使用

3. **衍生属性（核心）**
   - 衍生属性由基础属性计算得出
   - 不同的职业使用不同的计算公式

4. **生活技能（核心）**
   - 每个技能都是一个值
   - 这些值通常会作为判定系统中的加值使用
   - 基础的技能值属于属性值的衍生值
   - 技能值通常会作为判定系统的加值使用

5. **专长系统**
   - beta1版本不开发

6. **职业系统**
   - beta1版本只开发战士
   - 职业决定了冒险者的一系列的相关能力
   - 职业本质上决定了角色的实际构成
   - 可以理解为每个职业都是对基础属性、衍生属性、生活技能的各项值的再计算方式

7. **配置系统（核心）**
   - 用来配置职业、职业能力、职业的相关属性计算方式及各项包装内容的配置系统
   - 这部分优先开发

## AI模型对dnd规则的调用

### 模组注入
- 指将一个模组用特定的方式撰写，并导入到游戏中的过程
- 理想情况下，在相关功能开发后，AI可以智能的将模组1键进行注入，并直接生成相关的冒险模组
